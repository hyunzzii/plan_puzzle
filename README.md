## Commit rule 📃

### Name

- `태그: 커밋 제목`의 규칙으로 작성
- 작은 단위로 커밋을 작성하는 것을 기본으로 함

### Tag type

- `Init` : 프로젝트 생성
- `Feat` : 새로운 기능 추가
- `Chore` : 사소한 코드 수정
- `Fix` : 에러 및 버그 수정
- `Docs` : 문서 수정
- `Design` : 디자인 관련 코드 추가 및 수정
- `Refactor` : 코드 리팩토링
- `CI/CD` : 배포 관련 설정 추가 및 수정
